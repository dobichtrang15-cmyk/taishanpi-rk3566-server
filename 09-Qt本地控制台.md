# Qt 本地控制台

## 目标

把当前 HDMI 小屏控制台从浏览器页面改为本地 Qt 程序，避免浏览器缓存、缩放和小屏布局问题。

Qt 版本复用现有后端接口：

```text
GET  /api/auth/status
POST /api/login
POST /api/logout
GET  /api/device/workstation
GET  /api/device/workstation/status
POST /api/device/workstation/wake
POST /api/device/workstation/shutdown
GET  /api/files?path=/
GET  /api/syncthing/status
```

## 技术选择

当前实现使用：

```text
Python 3
PyQt5
requests
```

原因：

- Ubuntu focal 上安装简单。
- ARM 开发板兼容性更稳。
- 不再依赖浏览器窗口和 kiosk 参数。
- 更适合后续继续做本地触屏界面。

## 文件

Qt 程序：

```text
qt/kiosk_qt.py
```

安装脚本：

```text
scripts/install-qt-kiosk.sh
```

## 安装

先把文件传到开发板，例如：

```bash
scp kiosk_qt.py lckfb@192.168.50.1:/home/lckfb/
scp install-qt-kiosk.sh lckfb@192.168.50.1:/home/lckfb/
```

开发板执行：

```bash
cd /home/lckfb
chmod +x install-qt-kiosk.sh
sudo ./install-qt-kiosk.sh ./kiosk_qt.py
sudo reboot
```

## 界面结构

Qt 版本保留当前控制台结构：

- 登录界面
- 主页概览
- 设备控制
- 文件列表
- 同步状态

当前首页已经按小 HDMI 屏优化为单列布局：

- 顶部三项状态卡
- 中部 2x2 操作按钮
- 底部三个信息面板纵向排列

主页包括：

- 顶部标题和刷新 / 退出
- 唤醒电脑
- 关闭电脑
- 文件浏览
- 同步状态
- 登录 / 笔记本 / 同步状态卡片
- 三个信息面板

## 当前行为

- 登录成功后自动切到主页。
- 每 15 秒刷新一次设备在线状态。
- 唤醒 / 关机前会确认。
- 设备在线状态兼容 `online`、`oline`、嵌套返回和中文状态文本。

## 回滚到浏览器 kiosk

如果需要恢复浏览器版本，只要把自动启动项改回原来的浏览器脚本：

```bash
sudo nano /home/dashboard/.config/autostart/dashboard-kiosk.desktop
```

把 `Exec` 改回原来的浏览器启动器。
