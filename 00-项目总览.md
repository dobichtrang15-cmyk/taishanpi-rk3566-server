# 立创开发板 3566 改装服务器

## 项目目标

把 Linux 开发板改造成一个可长期运行的小型家庭服务器，实现以下能力：

- 局域网访问网页文件管理系统。
- 通过 Cloudflare Tunnel 暴露到公网访问。
- 使用网页登录和权限控制保护文件。
- 支持文件上传、下载、文件夹上传、目录浏览、PDF/图片/Markdown 预览。
- 支持控制笔记本电脑开机和关机。
- 开发板 WiFi 负责公网连接，有线网口直连笔记本，用于 Wake-on-LAN 和 SSH 控制。

## 当前结论

当前架构已经验证可用：

- `Nginx` 负责前端静态页面和反向代理。
- `Flask` 后端负责文件管理、登录权限、设备控制。
- `Cloudflared` 负责公网隧道。
- `eth0` 固定为 `192.168.50.1/24`，直连笔记本有线网口 `192.168.50.2/24`。
- 笔记本远程关机通过 SSH 执行 Windows 关机命令。
- 笔记本开机通过 Wake-on-LAN 魔术包唤醒。

| 对象         | 地址 / 配置                | 作用                   |

| 开发板 WiFi   | `192.168.2.209`        | 连接家庭网络和 Cloudflare   |

| Flask 后端   | `127.0.0.1:5000`       | API 服务               |
| Nginx 前端   | `127.0.0.1:80`         | Web 入口               |
|            |                        |                      |

## 目录结构

| 路径                                        | 作用              |     |
| ----------------------------------------- | --------------- | --- |
| `/userdata/server/www/site/index.html`    | 前端页面            |     |
| `/userdata/server/apps/filemgr/app.py`    | Flask 后端        |     |
| `/userdata/files`                         | 实际文件存储目录        |     |
| `/etc/systemd/system/filemgr.service`     | Flask 后端开机自启    |     |
| `/etc/systemd/system/eth0-direct.service` | eth0 直连 IP 开机配置 |     |
| `/etc/nginx/sites-enabled/default`        | Nginx 站点配置      |     |
