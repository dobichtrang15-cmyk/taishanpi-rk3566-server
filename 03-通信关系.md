# 通信关系

## 公网访问链路

```mermaid
sequenceDiagram
    participant User as 外网用户/手机
    participant CF as Cloudflare
    participant Tunnel as cloudflared
    participant Nginx as Nginx 80
    participant Flask as Flask 5000
    participant FS as /userdata/files

    User->>CF: 访问 https://files.jjpersonal.xyz
    CF->>Tunnel: 通过 Tunnel 转发请求
    Tunnel->>Nginx: http://localhost:80
    Nginx-->>User: 返回 index.html / 静态资源
    User->>Nginx: 请求 /api/files
    Nginx->>Flask: 反向代理到 127.0.0.1:5000
    Flask->>FS: 读取目录
    Flask-->>Nginx: JSON
    Nginx-->>User: JSON
```

## 文件上传链路

```mermaid
sequenceDiagram
    participant Browser as 浏览器
    participant Nginx as Nginx
    participant Flask as Flask
    participant FS as /userdata/files

    Browser->>Nginx: POST /api/upload
    Nginx->>Flask: 转发 multipart/form-data
    Flask->>Flask: 校验登录和上传权限
    Flask->>Flask: 校验目标路径安全
    Flask->>FS: 写入文件
    Flask-->>Browser: {"ok": true}
```

## 文件下载链路

```mermaid
sequenceDiagram
    participant Browser as 浏览器
    participant Nginx as Nginx
    participant Flask as Flask
    participant FS as /userdata/files

    Browser->>Nginx: GET /api/download?path=...
    Nginx->>Flask: 反向代理请求
    Flask->>Flask: 校验登录和下载权限
    Flask->>FS: 定位文件
    Flask-->>Nginx: 返回文件流或 X-Accel-Redirect
    Nginx-->>Browser: 下载文件
```

## 笔记本开机链路

```mermaid
sequenceDiagram
    participant Browser as 网页
    participant Flask as Flask
    participant Eth0 as 开发板 eth0 192.168.50.1
    participant NIC as 笔记本有线网卡 4C:CF:7C:DA:08:56
    participant Laptop as 笔记本

    Browser->>Flask: POST /api/device/workstation/wake
    Flask->>Eth0: 发送 WOL 魔术包
    Eth0->>NIC: 广播到 192.168.50.255
    NIC->>Laptop: 网卡识别 MAC 并触发上电
```

## 笔记本关机链路

```mermaid
sequenceDiagram
```

## 当前网络关系

| 网络 | 设备 | 地址 | 用途 |
|---|---|---|---|
| 家庭 WiFi | 开发板 wlan0 | `192.168.2.209/24` | Web、公网隧道 |
| 家庭 WiFi | 笔记本 WLAN | `192.168.2.92/24` | 普通上网 |
| 直连有线 | 开发板 eth0 | `192.168.50.1/24` | WOL/SSH 控制 |
| 直连有线 | 笔记本以太网 | `192.168.50.2/24` | 被控制端 |

## 关键判断

- 文件管理走 `Nginx -> Flask -> /userdata/files`。
- 公网访问走 `Cloudflare -> cloudflared -> Nginx`。
- 笔记本控制不走 WiFi，而走开发板和笔记本之间的有线直连网络。
- 网页显示 SSH 离线时，通常代表 `192.168.50.2:22` 不通。
- 笔记本关机后 SSH 离线是正常现象，但 WOL 仍然可以通过广播唤醒。
