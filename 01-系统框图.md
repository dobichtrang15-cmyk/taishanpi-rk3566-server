# 系统框图

## 总体架构

```mermaid
flowchart LR
    Phone["手机 / 外网浏览器"]
    PCBrowser["笔记本浏览器"]
    CF["Cloudflare DNS + Tunnel"]
    BoardWifi["开发板 wlan0\n192.168.2.209\n连接家庭 WiFi"]
    Nginx["Nginx\n80 端口\n静态页面 + 反向代理"]
    Flask["Flask filemgr\n127.0.0.1:5000\n业务 API"]
    Files["/userdata/files\n文件存储区"]
    BoardEth["开发板 eth0\n192.168.50.1/24"]
    LaptopEth["笔记本以太网\n192.168.50.2/24\nMAC 4C:CF:7C:DA:08:56"]
    LaptopOS["Windows\nOpenSSH Server\nWake-on-LAN"]

    Phone -->|"https://files.jjpersonal.xyz"| CF
    CF -->|"Tunnel 到 localhost:80"| BoardWifi
    PCBrowser -->|"http://192.168.2.209 或域名"| Nginx
    BoardWifi --> Nginx
    Nginx -->|"静态页面"| PCBrowser
    Nginx -->|"/api/* 反向代理"| Flask
    Flask -->|"读写文件"| Files
    Flask -->|"发送 magic packet"| BoardEth
    BoardEth -->|"192.168.50.255 广播"| LaptopEth
    Flask -->|"SSH 22 执行 shutdown"| LaptopEth
    LaptopEth --> LaptopOS
```

## 网络分层

```mermaid
flowchart TB
    Internet["公网访问层\nCloudflare Tunnel"]
    LanWifi["家庭 WiFi 层\n192.168.2.0/24"]
    DirectLink["设备控制直连层\n192.168.50.0/24"]
    AppLayer["应用服务层\nNginx + Flask"]
    StorageLayer["存储层\n/userdata/files"]

    Internet --> LanWifi
    LanWifi --> AppLayer
    AppLayer --> StorageLayer
    AppLayer --> DirectLink
```

## 关键设计

- 公网访问不做路由器端口转发，而是用 Cloudflare Tunnel。
- 文件系统不直接暴露给 Nginx 目录浏览，而是通过 Flask API 控制权限。
- 开发板 WiFi 和有线网口分工明确：WiFi 对外，有线对笔记本。
- Wake-on-LAN 不依赖家庭路由器广播，而是开发板通过直连网线向 `192.168.50.255` 广播。
